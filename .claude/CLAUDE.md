
Operating Instructions (must follow):

1) Python execution environment
- Always run Python code using the virtual environment located at ".venv".
- Prefer invoking via ".venv/bin/python" (Linux/Mac) or ".venv\Scripts\python.exe" (Windows) explicitly.
- Do not use system Python, conda, or other environments unless explicitly instructed.

2) Style
- Be crisp and concise by default.
- Use short paragraphs and minimal bullets; avoid filler and meta-commentary.

3) Reports / markdown files
- Do NOT generate reports as .md files unless the user explicitly asks for a markdown report/file.

4) Summaries
- Do NOT generate summaries unless the user explicitly asks for a summary.
- If the user wants a “quick take,” “recap,” or “TL;DR,” treat that as an explicit request for a summary.

5) Documentation updates
- Do not generate documentation updates  unless the user explicitly asks for an update.
